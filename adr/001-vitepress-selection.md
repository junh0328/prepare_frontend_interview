# ADR-001: 정적 사이트 생성기로 VitePress 선택

- **상태**: 승인됨
- **날짜**: 2025-11
- **결정자**: junhee lee

## 맥락

기존에 GitHub 마크다운으로만 제공되던 면접 준비 핸드북을 정적 웹사이트로 배포하여 접근성과 사용 경험을 개선하고자 했다. 다음 요구사항이 있었다:

- 기존 마크다운 파일을 최소한의 수정으로 재사용
- 한국어 전문 검색 기능
- 사이드바/네비게이션을 통한 주제별 탐색
- 빠른 페이지 로드 속도
- 무료 배포 (Vercel)

## 고려한 대안

| 도구                      | 장점                                                                   | 단점                                   |
| ------------------------- | ---------------------------------------------------------------------- | -------------------------------------- |
| **VitePress**             | Vite 기반 빠른 빌드, 마크다운 네이티브 지원, 로컬 검색 내장, 최소 설정 | Vue 기반 (React 생태계가 아님)         |
| **Docusaurus**            | React 기반, 다국어 지원, 플러그인 풍부                                 | 설정이 복잡, 빌드 속도 상대적으로 느림 |
| **GitBook**               | 마크다운 호환, 호스팅 포함                                             | 무료 플랜 제한, 커스터마이징 어려움    |
| **GitHub Pages + Jekyll** | GitHub 네이티브, 무료                                                  | 로컬 검색 없음, Ruby 의존성, 빌드 느림 |

## 결정

**VitePress**를 선택한다.

### 근거

1. **마크다운 친화성** — 기존 `.md` 파일을 거의 그대로 사용 가능. 이미지 경로만 변환하면 됨
2. **빌드 속도** — Vite 기반으로 6개 대형 마크다운 파일(총 13,000줄+)도 수초 내 빌드
3. **로컬 검색** — 별도 서비스 없이 한국어 전문 검색 가능 (`search.provider: 'local'`)
4. **최소 설정** — `config.js` 하나로 네비게이션, 사이드바, 검색, 테마 설정 완료
5. **Vercel 통합** — Framework Preset으로 원클릭 배포. 빌드 커맨드 자동 감지

### 트레이드오프

- Vue 기반이지만, 이 프로젝트는 커스텀 컴포넌트가 거의 없어 영향 미미
- Docusaurus의 풍부한 플러그인 생태계를 포기하지만, 현재 필요한 기능은 VitePress로 충분

## 결과

- `docs/.vitepress/config.js`에 사이트 설정 집중
- `scripts/sync-docs.js`로 루트 마크다운 → docs/ 자동 동기화 구현
- Vercel 자동 배포로 push마다 사이트 갱신
