# ADR-003: architecture.md 콘텐츠 포맷 결정

- **상태**: 승인됨
- **날짜**: 2026-02
- **결정자**: junhee lee

## 맥락

"아키텍처에 대해 고민이 있으신가요?"라는 면접 질문에 대한 답변 가이드를 추가하면서, 기존 Q&A 핸드북 형식(포맷 A)과 다른 형태가 필요했다. 아키텍처 질문은 정해진 정답이 없는 경험 기반 열린 질문이기 때문에, 기존 "질문 → 3-5줄 답변" 패턴이 적합하지 않았다.

## 고려한 대안

| 대안 | 장점 | 단점 |
| --- | --- | --- |
| A) 기존 Q&A 형식으로 변환 | 일관성 유지 | 답변 전략/흐름이 분절됨, 꼬리 질문 맥락 소실 |
| B) 답변 전략 가이드 형식 (채택) | 실전 답변 흐름 유지, 꼬리 질문 대비 포함 | 기존 파일과 포맷 차이 발생 |
| C) 별도 디렉토리로 분리 | 완전한 분리 | 과도한 구조 변경, 기존 탐색 흐름 파괴 |

## 결정

**포맷 B(답변 전략 가이드)를 채택하고, question_list.md처럼 기존 핸드북과 다른 성격의 파일로 위치시킨다.**

### 포맷 B 구조

1. **답변 예시** — 인용 블록(`>`)으로 실제 면접 답변 스크립트 제시
2. **꼬리 질문 대비** — 테이블 형태로 예상 질문과 답변 포인트 정리
3. **코드/구조 예시** — `<details>` 접기로 보충 자료 제공

### 적용 기준

- "X가 뭔가요?" (개념 설명, 정답 있음) → 포맷 A: Q&A 핸드북
- "어떻게 고민하셨나요?" (경험/전략, 정답 없음) → 포맷 B: 답변 전략 가이드

## 근거

1. 아키텍처 답변은 4개 레벨(시스템 → 앱 → 컴포넌트 → 코드)을 순차적으로 전개하는 **흐름**이 중요하며, Q&A 형식으로 쪼개면 이 흐름이 사라진다
2. 꼬리 질문이 답변과 밀접하게 연관되어 테이블로 함께 보여주는 것이 학습 효과가 높다
3. question_list.md도 기존 핸드북과 다른 형식(질문 목록 + 체크리스트)을 사용하는 선례가 있다

## 트레이드오프

- 두 가지 포맷 공존으로 CLAUDE.md에 포맷 선택 기준 문서화 필요 (완료)
- 향후 경험 기반 질문이 추가될 때 포맷 B 사용 여부를 매번 판단해야 함

## 결과

- architecture.md가 포맷 B로 생성됨 (PR #38)
- CLAUDE.md에 "콘텐츠 구성 패턴" 섹션에서 두 포맷의 사용 기준 명시
- architecture.md ↔ react.md/cs.md 간 크로스 링크 추가
