# 프론트엔드 면접 핸드북 고도화 작업 계획

## 작업 개요

CLAUDE.md의 가이드라인을 바탕으로 기존 마크다운 파일들의 품질을 개선하고 일관성 있는 구조로 재정비합니다.

## 작업 대상 파일 (우선순위 순)

### 1. js.md (최우선 - 질문 빈도 1위)
- 34개 주제 섹션
- 변수, 데이터 타입, 클로저, 프로미스, async/await 등

### 2. cs.md (높음 - 질문 빈도 2위)
- 15개 주제 섹션
- 네트워킹, HTTP, 운영체제, 프로세스/스레드, 쿠키/세션, CORS, 웹팩 등

### 3. react.md (높음 - 질문 빈도 3위)
- 8개 주요 섹션 (일부 구조화 완료)
- React 기초, 파이버, 훅, 상태 관리, Next.js, Suspense 등

### 4. html_css.md (중간 - 질문 빈도 4위)
- HTML과 CSS 섹션
- DOCTYPE, 웹 표준, display, position, flexbox, grid 등

### 5. data_structure.md (중간)
- 자료구조 기본 개념
- 리스트, 큐, 스택, 해시 테이블, 트리, 힙, 그래프

### 6. algorithm_data_structure.md (낮음)
- 알고리즘 패턴과 상세 구현
- 정렬, 탐색, 자료구조 구현

## 평가 기준 (5점 만점)

### 1. 구조화 (1.0점)
- [ ] 논리적인 주요 섹션(##) 구분
- [ ] 각 주요 섹션에 📌 관련 주제 링크 추가
- [ ] 목차 정리 및 앵커 링크 검증

### 2. 답변 품질 (1.5점)
- [ ] 답변을 3-5줄 핵심 내용으로 요약
- [ ] 40줄 이상 긴 답변은 `<details>` 태그로 접기 처리
- [ ] 기술 키워드 강조 및 명확화

### 3. 일관성 (1.0점)
- [ ] 답변 길이 균형
- [ ] 불 이모지(🔥) 사용 일관성
- [ ] 마크다운 형식 통일

### 4. 정확성 (1.0점)
- [ ] 오타 및 문법 오류 수정
- [ ] 기술적 내용 검증
- [ ] 링크 동작 확인

### 5. 완성도 (0.5점)
- [ ] 누락된 섹션 확인

## 주요 작업 내용

### 🔴 높은 우선순위 (필수)
1. **섹션 재구조화**
   - 관련 개념별로 논리적 그룹핑
   - 명확한 주요 섹션(##) 구분

2. **관련 주제 링크 추가**
   - 각 주요 섹션마다 📌 관련 주제 2-4개 추가
   - 형식: `📌 **관련 주제**: [섹션1](#앵커), [섹션2](#앵커)`

3. **답변 길이 불균형 해결**
   - 핵심 답변: 3-5줄로 요약
   - 긴 내용: `<details>` 태그로 접기 처리

4. **명백한 오타 수정**
   - 기술 용어 표기 통일
   - 맞춤법 및 문법 오류 수정

### 🟡 중간 우선순위 (권장)
5. **목차 정리**
   - 파일 내부 목차 업데이트
   - README.md 목차 동기화

6. **답변 개선**
   - 핵심 키워드 누락 보완
   - 기술 용어 명확화

7. **예제 코드 추가**
   - 설명이 부족한 개념 보강

### 🟢 낮은 우선순위 (선택)
8. **스타일 통일**
   - 마크다운 형식 일관성

9. **이미지 추가**
   - 시각적 설명 필요한 개념

10. **추가 질문**
